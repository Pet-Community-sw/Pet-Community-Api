이번에 한 일
- 단순 조회/위임/매퍼 성격의 테스트를 대거 제거하고, 회귀 위험이 큰 테스트만 남겼다.
- 남긴 테스트는 위치 파이프라인, 채팅 재전송/읽음 처리, 댓글 회귀, 산책 상태 전이, Redis 값 파싱, 채팅방 프로필 배치 조회 검증이다.
- `./gradlew test`로 전체 테스트를 다시 실행해 통과를 확인했다.

현재 진행 상태
- 의미 있는 테스트만 남기도록 테스트 스위트 정리가 완료되었다.
- 현재 `src/test/java`에는 핵심 로직을 검증하는 테스트 8개만 남아 있다.

다음에 할 일
- 남겨둔 핵심 테스트가 실제 버그를 계속 잘 막는지, 기능 수정 시 테스트 이름과 의도를 함께 유지한다.
- 자동완성 캐시 역직렬화 이슈처럼 아직 분석만 끝난 항목은 별도 수정 작업으로 진행한다.

이번 세션 추가 분석
- TDD 전환을 위해 현재 테스트 공백을 다시 검토했고, 우선순위가 높은 테스트 후보를 정리했다.
- 최우선 후보는 `ProfileService.updateProfile`의 기피 견종 갱신 회귀, `ScheduleService.getSchedules`의 profile 미존재 경로, `WalkRecordService.finishWalkRecord`의 경로/거리/정리 처리, `TokenService`의 재발급·로그아웃 블랙리스트 처리, STOMP 구독 권한 검증이다.
- 이미 남아 있는 위치 파이프라인, 채팅 재전송, 읽음 처리, Redis 파싱 테스트는 그대로 유지하면 된다.

이번 세션 추가 작업
- `LocationPipeline`에서 `BackpressureStrategy.LATEST`를 제거하고 `Observable` 체인으로 단순화했다.
- `throttleFirst` 기반 이벤트 제한은 유지한 상태로 `LocationPipelineTest`, 전체 `./gradlew test` 통과를 확인했다.

이번 세션 추가 작업
- `LocationPipeline`의 timeout 및 파이프 에러를 `subscribe`의 에러 consumer에서 로그 후 `clean()` 하도록 단순화했다.
- `doOnError`, `onErrorComplete`, `onComplete` 기반 정리를 제거하고 `LocationPipelineTest`, 전체 `./gradlew test` 통과를 확인했다.
